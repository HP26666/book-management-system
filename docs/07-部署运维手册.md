## 1. 引言

### 1.1 编写目的

本文档指导开发、测试、运维人员使用 Docker 部署图书管理系统，并说明环境变量、健康检查、日志、备份、发布和故障排查要求。

### 1.2 部署边界

| 模块 | 是否 Docker 部署 | 说明 |
|------|------------------|------|
| PostgreSQL | 是 | Docker Compose 编排 |
| Redis | 是 | Docker Compose 编排 |
| Spring Boot 后端 | 是 | `book-management-backend/Dockerfile` 构建 |
| Vue 管理端 | 是 | `book-management-admin/Dockerfile` 构建，Nginx 托管静态资源 |
| Uni-App 小程序端 | 否 | 使用 HBuilderX/微信开发者工具构建发布 |

## 2. 环境准备

### 2.1 服务器建议配置

| 环境 | CPU | 内存 | 磁盘 | 说明 |
|------|-----|------|------|------|
| 开发环境 | 4 核 | 8 GB | 100 GB | 单机 Docker Compose |
| 测试环境 | 4 核 | 8 GB | 100 GB | 单机 Docker Compose |
| 生产环境 | 8 核 | 16 GB | 500 GB | 可先单机，后续拆分数据库与应用 |

### 2.2 必备软件

| 软件 | 版本建议 | 用途 |
|------|----------|------|
| Docker Engine | 24+ | 容器运行 |
| Docker Compose Plugin | 2.20+ | 服务编排 |
| Git | 2.40+ | 拉取代码 |
| JDK | 17 LTS | 本地后端开发 |
| Node.js | 20 LTS | 本地管理端、小程序端开发 |
| Maven | 3.9+ | 本地后端构建 |

服务器部署只强依赖 Docker 与 Docker Compose；JDK、Node.js、Maven 主要用于非容器本地开发。

## 3. 环境变量

根目录提供 `.env.example`。首次部署：

```bash
cp .env.example .env
```

关键变量：

| 变量 | 默认值 | 说明 |
|------|--------|------|
| `POSTGRES_DB` | `book_management` | 数据库名 |
| `POSTGRES_USER` | `bookuser` | 数据库用户 |
| `POSTGRES_PASSWORD` | 无安全默认值 | 必须在测试/生产修改 |
| `POSTGRES_PORT` | `5432` | 宿主机暴露端口 |
| `REDIS_PORT` | `6379` | 宿主机暴露端口 |
| `BACKEND_PORT` | `8080` | 后端宿主机端口 |
| `ADMIN_PORT` | `80` | 管理端宿主机端口 |
| `JWT_SECRET` | 无安全默认值 | 生产必须使用 32 位以上随机字符串 |

安全要求：

1. `.env` 不提交版本库。
2. 生产环境不得使用 `.env.example` 中的示例密码。
3. `JWT_SECRET` 泄漏后必须立即轮换，并清理已签发 token。

## 4. Docker Compose 部署

### 4.1 服务组成

当前 `docker-compose.yml` 包含：

| 服务 | 容器名 | 端口 | 健康检查 |
|------|--------|------|----------|
| `postgres` | `book-postgres` | `${POSTGRES_PORT}:5432` | `pg_isready` |
| `redis` | `book-redis` | `${REDIS_PORT}:6379` | `redis-cli ping` |
| `backend` | `book-backend` | `${BACKEND_PORT}:8080` | `/actuator/health` |
| `admin` | `book-admin` | `${ADMIN_PORT}:80` | 依赖后端健康 |

### 4.2 启动

```bash
docker compose up -d --build
```

### 4.3 查看状态

```bash
docker compose ps
docker compose logs -f backend
docker compose logs -f admin
```

### 4.4 停止与清理

```bash
docker compose down
```

如需清空数据库和 Redis 数据卷：

```bash
docker compose down -v
```

执行 `down -v` 会删除数据，只允许在开发或明确确认的测试环境使用。

## 5. 本地开发运行

### 5.1 后端

```bash
cd book-management-backend
mvn spring-boot:run
```

本地默认连接：

```text
jdbc:postgresql://localhost:5432/book_management
Redis localhost:6379
```

### 5.2 管理端

```bash
cd book-management-admin
npm install
npm run dev
```

Vite 默认端口 `5173`，`/api` 代理到 `http://localhost:8080`。

### 5.3 小程序端

```bash
cd book-management-uniapp
npm install
npm run dev:mp-weixin
```

也可使用 HBuilderX 导入项目后运行到微信开发者工具。小程序发布前必须在微信公众平台配置合法 request 域名，生产接口必须使用 HTTPS。

## 6. Nginx 与访问路径

管理端容器内 Nginx 配置位于 `book-management-admin/nginx.conf`：

| 路径 | 说明 |
|------|------|
| `/` | Vue 管理端静态资源，支持 history fallback |
| `/api/` | 反向代理到 `book-backend:8080/api/` |
| `/actuator/` | 反向代理后端健康检查等运维接口 |

生产环境建议：

1. 使用独立域名，如 `admin.example.com`。
2. 在外层网关或云负载均衡配置 HTTPS。
3. 限制 `/actuator/` 访问来源，避免暴露过多运维信息。

## 7. 数据库初始化与迁移

当前框架尚未加入迁移工具。后续开发必须补充 Flyway 或 Liquibase，建议：

```text
book-management-backend/src/main/resources/db/migration/
├── V1__init_schema.sql
├── V2__init_seed_data.sql
└── V3__add_xxx.sql
```

规则：

1. 已发布迁移脚本禁止重写，只能追加新版本。
2. 初始化脚本必须与《04-数据库设计说明书》保持一致。
3. 测试环境每次发布后必须执行迁移验证。
4. 生产迁移前必须备份数据库。

## 8. 日志与监控

| 对象 | 查看方式 |
|------|----------|
| 后端日志 | `docker compose logs -f backend` |
| 管理端 Nginx 日志 | `docker compose logs -f admin` |
| PostgreSQL 日志 | `docker compose logs -f postgres` |
| Redis 日志 | `docker compose logs -f redis` |

基础健康检查：

```bash
curl http://localhost:8080/actuator/health
curl http://localhost/actuator/health
```

后续生产增强建议：

1. 引入 Prometheus + Grafana 监控 JVM、接口耗时、数据库连接池、Redis。
2. 接入集中日志平台，按 traceId 串联前后端请求。
3. 对登录失败、库存扣减失败、接口 5xx、数据库连接失败配置告警。

## 9. 备份与恢复

### 9.1 数据库备份

```bash
docker exec book-postgres pg_dump -U bookuser -d book_management > backup_$(date +%Y%m%d).sql
```

生产要求：

1. 每日至少一次全量备份。
2. 备份文件保留 30 天以上。
3. 备份文件应加密并异地保存。
4. 每月至少进行一次恢复演练。

### 9.2 文件备份

若后续图书封面使用本地挂载目录存储，必须将挂载目录纳入备份；生产更推荐使用对象存储，如 MinIO、阿里云 OSS、腾讯云 COS。

## 10. 发布流程

1. 确认代码已合并到发布分支。
2. 确认接口文档、数据库文档、测试用例已同步更新。
3. 执行后端单元测试、管理端构建、小程序构建。
4. 备份目标环境数据库。
5. 执行 `docker compose up -d --build`。
6. 检查 `docker compose ps` 所有服务健康。
7. 执行验收冒烟用例：登录、图书查询、借阅申请、审批、归还、预约。
8. 记录版本号、发布时间、变更内容和回滚方案。

---

## 11. 回滚流程

1. 保留上一版本镜像或代码标签。
2. 如仅应用异常，回滚后端/管理端镜像并重启。
3. 如数据库迁移已执行，优先通过兼容迁移修复；禁止随意手工删列或改历史脚本。
4. 如数据损坏，按备份恢复流程处理，并记录影响范围。

---

## 12. 常见问题排查

| 现象 | 排查方向 |
|------|----------|
| 后端无法启动 | 查看 `docker compose logs backend`，重点检查数据库连接、环境变量、端口占用 |
| 管理端页面空白 | 查看浏览器控制台、Nginx 日志、构建产物是否存在 |
| `/api` 访问失败 | 检查 `book-management-admin/nginx.conf` 代理、后端容器健康状态 |
| 数据库连接失败 | 检查 `POSTGRES_*` 变量、PostgreSQL 健康检查、数据卷权限 |
| 小程序无法请求接口 | 检查 HTTPS、微信合法域名、后端 CORS、接口证书 |
| 借阅并发库存异常 | 检查库存扣减 SQL 是否为条件更新，是否有事务和测试覆盖 |

## 13. Agent 开发要求

后续 agent 修改部署相关内容时必须同步检查：

1. `docker-compose.yml`、后端 `application.yml`、管理端 `nginx.conf` 的服务名和端口是否一致。
2. README 的快速启动命令是否仍然可用。
3. 新增环境变量是否写入 `.env.example` 并在本手册说明。
4. 新增存储组件是否有数据卷、备份策略、健康检查和清理说明。
5. 小程序端不加入 Docker 运行链路，但必须保留独立构建和发布说明。
