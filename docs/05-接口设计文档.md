## 2. 接口规范

### 2.1 通信协议

- 协议：HTTP/HTTPS
- 数据格式：JSON（Content-Type: application/json）
- 字符编码：UTF-8

### 2.2 认证方式

- 采用 **Bearer Token** 认证机制
- 登录成功后，服务端返回 `accessToken` 和 `refreshToken`
- 客户端在后续请求的 `Authorization` 头部携带：`Bearer {accessToken}`
- Token 过期后使用 `refreshToken` 换取新的 `accessToken`

### 2.3 统一响应格式

```json
{
  "code": 200,
  "message": "success",
  "data": {},
  "traceId": "optional-trace-id"
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| code | Integer | 业务状态码，200为成功 |
| message | String | 提示信息 |
| data | Object | 响应数据，可为对象、数组或null |
| traceId | String | 可选，链路追踪ID；生产排障时建议返回 |

分页接口统一使用以下 `data` 结构：

```json
{
  "total": 100,
  "pages": 10,
  "current": 1,
  "size": 10,
  "records": []
}
```

### 2.4 全局错误码定义

| 错误码 | 含义 | 说明 |
|--------|------|------|
| 200 | 成功 | 请求处理成功 |
| 400 | 请求参数错误 | 参数校验失败或格式不正确 |
| 401 | 未授权 | Token无效或已过期 |
| 403 | 禁止访问 | 权限不足 |
| 404 | 资源不存在 | 请求的接口或数据不存在 |
| 409 | 业务冲突 | 如库存不足、重复操作等 |
| 500 | 服务器内部错误 | 系统异常 |

## 4. 接口详表

### 4.1 认证接口

#### POST /api/auth/login

**请求参数：**

```json
{
  "username": "admin",
  "password": "123456"
}
```

**响应示例：**

```json
{
  "code": 200,
  "message": "登录成功",
  "data": {
    "accessToken": "eyJhbGciOiJIUzI1NiIs...",
    "refreshToken": "dGhpcyBpcyBhIHJlZnJlc2g...",
    "expiresIn": 7200,
    "user": {
      "id": 1,
      "username": "admin",
      "realName": "管理员",
      "roles": ["admin"]
    }
  }
}
```

---

**请求参数：**

```json
{
  "username": "zhangsan",
  "password": "123456",
  "realName": "张三",
  "phone": "13800138000"

```

**响应示例：**

```json
{
  "code": 200,
  "message": "注册成功",
  "data": {
    "id": 2,
    "username": "zhangsan"
  }
```

**请求参数：**

```json
{
  "code": "wx-login-code",
  "encryptedData": "optional",
  "iv": "optional",
  "phoneCode": "optional"
```

**响应示例：**


## 6. Agent 接口开发约束

1. 所有业务接口路径必须以 `/api` 开头；健康检查、OpenAPI 文档等运维接口除外。
2. 新增接口必须同步补充请求参数、响应示例、权限要求和异常场景。
3. 管理端接口默认需要管理员或图书管理员角色；小程序端只允许访问本人数据，后端必须校验 `userId` 归属。
4. 对外返回不得包含密码、Token 密钥、数据库连接信息等敏感字段。
5. 删除接口默认执行逻辑删除；确需物理删除必须在接口说明中写明原因。
6. 库存、借阅、预约接口必须覆盖并发冲突响应，冲突统一返回 `409`。
7. 文件上传必须限制类型、大小和存储路径，禁止使用用户原始文件名直接落盘。
