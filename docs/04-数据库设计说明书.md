## 1. 引言

### 1.1 编写目的

本文档描述图书管理系统 PostgreSQL 数据库的逻辑结构、物理结构、索引、枚举和初始化要求，为后续自动开发、迁移脚本编写、测试数据准备和运维备份提供依据。

### 1.2 重要修订说明

上一版表结构示例混用了 MySQL 风格的列内 `COMMENT`，在 PostgreSQL 中不能直接执行。本版统一采用 PostgreSQL 语法：建表语句中不写列内 `COMMENT`，如需注释使用 `COMMENT ON TABLE/COLUMN`；模糊搜索索引使用 `pg_trgm` 扩展；视图引用字段必须在基础表中真实存在。

## 3. 命名与通用字段规范

| 对象类型 | 命名规则 | 示例 |
|----------|----------|------|
| 表名 | 小写下划线，按领域前缀分组 | `sys_user`, `book_info` |
| 字段名 | 小写下划线 | `created_at`, `user_id` |
| 主键 | `pk_表名`，简单场景可使用默认主键名 | `pk_sys_user` |
| 外键 | `fk_表名_字段名` | `fk_book_info_category_id` |
| 索引 | `idx_表名_字段名` | `idx_book_info_isbn` |
| 唯一索引 | `uk_表名_字段名` | `uk_sys_user_username` |

通用字段要求：

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | `BIGSERIAL` | 自增主键 |
| `created_at` | `TIMESTAMP` | 创建时间 |
| `updated_at` | `TIMESTAMP` | 更新时间 |
| `deleted` | `BOOLEAN` | 逻辑删除标记，核心业务表必须具备 |

时间字段由应用层统一维护。需要数据库自动更新时间时，可在迁移脚本中增加触发器，但业务规则仍以应用层为主。

## 4. 实体关系设计

```text
[sys_user] N -- M [sys_role] N -- M [sys_permission]
[sys_user] 1 -- 1 [reader_info]
[sys_user] 1 -- N [borrow_record]
[sys_user] 1 -- N [reservation_record]
[book_category] 1 -- N [book_info]
[book_info] 1 -- N [borrow_record]
[book_info] 1 -- N [reservation_record]
[book_info] 1 -- N [book_stock_log]
```

设计约束：

1. 用户与角色、角色与权限采用中间表维护。
2. 读者信息通过 `reader_info.user_id` 与系统用户一对一绑定。
3. 图书库存采用 `total_stock` 与 `available_stock` 双字段；是否可借由 `available_stock > 0` 和 `status = 1` 共同决定。
4. 借阅审批通过时扣减可借库存；归还时释放库存；所有库存变化必须记录到 `book_stock_log`。
5. 预约只在无可借库存时开放，图书归还后按预约时间顺序通知。

## 5. 初始化扩展

```sql
CREATE EXTENSION IF NOT EXISTS pg_trgm;
```

`pg_trgm` 用于图书标题、作者等字段的模糊搜索。若生产数据库账号没有创建扩展权限，应由 DBA 在数据库初始化阶段执行。

## 6. 物理结构设计

### 6.1 用户权限相关表

```sql
CREATE TABLE sys_user (
    id BIGSERIAL PRIMARY KEY,
    username VARCHAR(50) NOT NULL,
    password VARCHAR(255) NOT NULL,
    real_name VARCHAR(50),
    phone VARCHAR(20),
    email VARCHAR(100),
    avatar_url VARCHAR(255),
    status SMALLINT NOT NULL DEFAULT 1,
    last_login_at TIMESTAMP,
    created_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    deleted BOOLEAN NOT NULL DEFAULT FALSE
);

CREATE UNIQUE INDEX uk_sys_user_username ON sys_user(username) WHERE deleted = FALSE;
CREATE INDEX idx_sys_user_phone ON sys_user(phone);
CREATE INDEX idx_sys_user_status ON sys_user(status);
```

```sql
CREATE TABLE sys_role (
    id BIGSERIAL PRIMARY KEY,
    role_code VARCHAR(50) NOT NULL UNIQUE,
    role_name VARCHAR(50) NOT NULL,
    description VARCHAR(255),
    status SMALLINT NOT NULL DEFAULT 1,
    created_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE sys_user_role (
    user_id BIGINT NOT NULL,
    role_id BIGINT NOT NULL,
    PRIMARY KEY (user_id, role_id),
    CONSTRAINT fk_sys_user_role_user_id FOREIGN KEY (user_id) REFERENCES sys_user(id),
    CONSTRAINT fk_sys_user_role_role_id FOREIGN KEY (role_id) REFERENCES sys_role(id)
);
```

```sql
CREATE TABLE sys_permission (
    id BIGSERIAL PRIMARY KEY,
    perm_code VARCHAR(100) NOT NULL UNIQUE,
    perm_name VARCHAR(100) NOT NULL,
    type VARCHAR(20) NOT NULL,
    parent_id BIGINT,
    sort_order INT NOT NULL DEFAULT 0,
    path VARCHAR(255),
    icon VARCHAR(100),
    status SMALLINT NOT NULL DEFAULT 1,
    created_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE sys_role_permission (
    role_id BIGINT NOT NULL,
    perm_id BIGINT NOT NULL,
    PRIMARY KEY (role_id, perm_id),
    CONSTRAINT fk_role_perm_role_id FOREIGN KEY (role_id) REFERENCES sys_role(id),
    CONSTRAINT fk_role_perm_perm_id FOREIGN KEY (perm_id) REFERENCES sys_permission(id)
);
```

### 6.2 图书分类与图书信息表

```sql
CREATE TABLE book_category (
    id BIGSERIAL PRIMARY KEY,
    name VARCHAR(100) NOT NULL,
    parent_id BIGINT,
    sort_order INT NOT NULL DEFAULT 0,
    level INT NOT NULL DEFAULT 1,
    created_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    deleted BOOLEAN NOT NULL DEFAULT FALSE,
    CONSTRAINT fk_book_category_parent_id FOREIGN KEY (parent_id) REFERENCES book_category(id)
);

CREATE INDEX idx_book_category_parent_id ON book_category(parent_id);
```

```sql
CREATE TABLE book_info (
    id BIGSERIAL PRIMARY KEY,
    isbn VARCHAR(20),
    title VARCHAR(255) NOT NULL,
    author VARCHAR(255),
    publisher VARCHAR(100),
    publish_date DATE,
    category_id BIGINT,
    cover_url VARCHAR(500),
    summary TEXT,
    price NUMERIC(10,2),
    total_stock INT NOT NULL DEFAULT 0,
    available_stock INT NOT NULL DEFAULT 0,
    status SMALLINT NOT NULL DEFAULT 1,
    location VARCHAR(100),
    created_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    deleted BOOLEAN NOT NULL DEFAULT FALSE,
    CONSTRAINT fk_book_info_category_id FOREIGN KEY (category_id) REFERENCES book_category(id),
    CONSTRAINT ck_book_stock_non_negative CHECK (total_stock >= 0 AND available_stock >= 0),
    CONSTRAINT ck_book_available_not_exceed_total CHECK (available_stock <= total_stock)
);

CREATE UNIQUE INDEX uk_book_info_isbn ON book_info(isbn) WHERE isbn IS NOT NULL AND deleted = FALSE;
CREATE INDEX idx_book_info_category_id ON book_info(category_id);
CREATE INDEX idx_book_info_status ON book_info(status);
CREATE INDEX idx_book_info_title_trgm ON book_info USING gin(title gin_trgm_ops);
CREATE INDEX idx_book_info_author_trgm ON book_info USING gin(author gin_trgm_ops);
```

`book_info.status` 只表示业务上下架状态：`0-下架`、`1-上架`。库存是否全部借出不写入状态字段，由 `available_stock = 0` 判断。

## 8. 视图设计

```sql
CREATE VIEW v_book_borrow_stat AS
SELECT
    b.id AS book_id,
    b.title,
    b.isbn,
    COUNT(br.id) AS total_borrow_count,
    SUM(CASE WHEN br.status = 2 THEN 1 ELSE 0 END) AS current_borrow_count
FROM book_info b
LEFT JOIN borrow_record br ON b.id = br.book_id AND br.deleted = FALSE
WHERE b.deleted = FALSE
GROUP BY b.id, b.title, b.isbn;
```

```sql
CREATE VIEW v_user_borrow_summary AS
SELECT
    u.id AS user_id,
    u.username,
    u.real_name,
    COUNT(br.id) AS total_borrows,
    SUM(CASE WHEN br.status = 2 THEN 1 ELSE 0 END) AS active_borrows,
    SUM(CASE WHEN br.status = 5 THEN 1 ELSE 0 END) AS overdue_borrows
FROM sys_user u
LEFT JOIN borrow_record br ON u.id = br.user_id AND br.deleted = FALSE
WHERE u.deleted = FALSE
GROUP BY u.id, u.username, u.real_name;
```

## 9. 迁移脚本与 Agent 开发要求

1. 首个迁移脚本建议命名为 `V1__init_schema.sql`，路径建议为 `book-management-backend/src/main/resources/db/migration/`。
2. 每次新增字段、索引、枚举值必须新增迁移脚本，不允许直接修改已发布脚本。
3. 后端 Entity 字段名、DTO 字段名、接口文档字段名必须与本数据库文档保持可追溯关系。
4. 删除业务数据默认采用逻辑删除；物理删除仅允许用于中间表或明确无审计价值的数据。
5. 库存扣减必须使用数据库条件更新或乐观锁，禁止先查库存再无保护地更新库存。
6. 测试数据应覆盖管理员、图书管理员、普通读者、无库存图书、逾期借阅、预约队列等场景。

## 10. 备份与恢复策略

| 项目 | 要求 |
|------|------|
| 全量备份 | 每日至少一次 `pg_dump`，生产建议保留 30 天 |
| 恢复演练 | 每月至少一次在非生产库验证备份可恢复 |
| 恢复目标 | RPO 不超过 24 小时，RTO 不超过 4 小时 |
| 敏感数据 | 测试环境恢复生产备份前必须脱敏手机号、邮箱、姓名等信息 |
