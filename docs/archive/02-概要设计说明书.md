## 4. 接口设计

### 4.1 外部接口

- **微信登录接口**：调用微信 OAuth2.0 接口获取用户 openid 和 unionid
- **ISBN查询接口**：对接第三方图书元数据服务（如豆瓣读书API）
- **文件存储接口**：可选对接阿里云OSS或自建MinIO服务

### 4.2 内部接口

- 后端各模块间通过 Service 层接口调用
- 前端与后端通过 RESTful API 进行 JSON 数据交互
- 统一响应格式：`{ "code": 200, "message": "success", "data": {} }`
- 统一分页响应格式：`{ "total": 0, "pages": 0, "current": 1, "size": 10, "records": [] }`
- 管理端和小程序端都只访问 `/api` 前缀，具体后端地址由代理或运行配置提供

## 6. 出错处理设计

- **全局异常捕获**：通过 Spring Boot 的 `@ControllerAdvice` 统一处理业务异常和系统异常
- **错误码体系**：定义统一错误码，如 `200` 成功、`400` 参数错误、`401` 未授权、`403` 无权限、`500` 系统错误
- **前端友好提示**：异常信息返回给前端，由前端组件统一展示错误提示

---

## 7. 安全设计

- **认证机制**：基于 JWT 的无状态认证，登录成功后发放 Access Token 和 Refresh Token
- **权限控制**：基于 Spring Security + RBAC 模型，实现接口级别的权限拦截
- **密码安全**：用户密码采用 BCrypt 算法加密存储
- **传输安全**：生产环境强制启用 HTTPS
- **输入校验**：全局参数校验，防止 SQL 注入和 XSS 攻击
- **操作审计**：关键操作记录操作日志，便于追溯

---

## 8. 维护设计

- **配置外部化**：数据库连接、Redis配置、JWT密钥等通过 `application.yml` 和外部配置文件管理
- **日志分级**：使用 SLF4J + Logback，区分 DEBUG、INFO、WARN、ERROR 级别
- **代码规范**：遵循阿里巴巴 Java 开发手册和 Vue 风格指南
- **版本控制**：使用 Git 进行代码版本管理，主干开发 + 功能分支模式
- **文档同步**：代码变更时同步更新接口文档（基于 Swagger / SpringDoc）

---

## 9. Agent 开发边界

1. 后续开发必须保持“Controller 只做协议适配，Service 处理业务，Repository 负责数据访问”的分层边界。
2. 涉及库存、借阅、预约的流程必须由后端保证事务一致性，前端只负责展示与触发。
3. PC 管理端与小程序端可以复用接口，但前端页面、权限入口、交互流程应分别设计。
4. 新增跨模块功能时必须同时检查需求、接口、数据库、测试、部署文档是否需要更新。
5. 当前不引入微服务拆分；除非验收目标全部完成，不新增网关、消息队列、搜索引擎等额外复杂组件。
