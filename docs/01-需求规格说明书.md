## 1. 引言

### 1.1 编写目的

本文档旨在明确图书管理系统的功能需求、性能需求及其他约束条件，为系统的设计、开发、测试和验收提供依据。目标读者包括项目经理、系统架构师、开发工程师、测试工程师及相关业务人员。

### 1.2 项目背景

随着数字化校园/企业建设的推进，传统的手工图书管理方式已无法满足日益增长的图书借阅需求。本项目旨在构建一套基于现代化技术栈的图书管理系统，支持PC Web端管理后台和微信小程序用户端，实现图书资源的高效管理与便捷借阅服务。

### 1.3 术语定义

| 术语 | 说明 |
|------|------|
| RBAC | 基于角色的访问控制（Role-Based Access Control） |
| JWT | JSON Web Token，一种无状态的身份认证机制 |
| RESTful | 一种基于HTTP协议的API设计风格 |
| Uni-App | DCloud推出的跨端应用开发框架 |
| ISBN | 国际标准书号（International Standard Book Number） |

### 1.4 参考资料

- GB/T 8567-2006 计算机软件文档编制规范

## 2. 任务概述

### 2.1 目标

建设一套完整的图书管理系统，实现以下核心目标：
1. 图书信息的数字化管理与检索
2. 借阅流程的线上化、自动化
3. 多终端覆盖（PC管理端 + 微信小程序用户端）
4. 数据统计与可视化分析

### 2.2 用户特点

| 用户角色 | 特点描述 |
|----------|----------|
| 系统管理员 | 具备最高权限，负责用户、角色、系统参数的配置 |
| 图书管理员 | 负责图书入库、分类维护、借阅审批、归还登记 |
| 普通读者 | 通过小程序查询图书、提交借阅申请、查看个人借阅记录 |

### 2.3 假定和约束

- 系统运行于局域网或互联网环境
- 微信小程序端需依赖微信生态登录体系
- 图书ISBN信息可通过第三方接口自动获取；第三方不可用时必须允许人工录入
- 服务器资源有限，需考虑部署成本
- PC Web 管理端和后端必须支持 Docker 部署，小程序端不纳入 Docker 运行
- 生产环境接口必须使用 HTTPS，微信小程序端必须配置合法 request 域名

## 3. 功能需求

### 3.1 功能模块划分

```
图书管理系统
├── 用户权限模块
├── 图书管理模块
├── 分类管理模块
├── 借阅管理模块
├── 读者管理模块
├── 预约管理模块
├── 公告/资讯模块
└── 系统管理模块
```

### 3.2 管理端功能需求

#### 3.2.1 用户与权限管理
- 用户信息的增删改查
- 角色定义与权限分配
- 用户状态启用/禁用

#### 3.2.2 图书管理
- 图书信息的录入、编辑、删除
- 支持ISBN自动解析图书元数据
- 图书封面上传与管理
- 库存数量维护
- 图书上下架状态切换

#### 3.2.3 借阅管理
- 查看所有借阅申请
- 借阅审批（通过/拒绝）
- 归还登记
- 逾期记录查询
- 罚金记录管理

#### 3.2.4 数据统计
- 图书借阅量统计
- 热门图书排行
- 读者借阅活跃度
- 数据可视化展示

### 3.3 用户端功能需求

#### 3.3.1 用户中心
- 微信授权登录
- 个人信息查看与修改
- 借阅证信息展示

#### 3.3.2 图书检索
- 关键词搜索图书
- 按分类筛选图书
- 扫码查书（扫描ISBN条码）
- 图书详情查看

#### 3.3.3 借阅与预约
- 在线提交借阅申请
- 查看借阅进度与历史
- 图书预约（无库存时）
- 续借申请

## 4. 非功能需求

### 4.1 性能需求

- 系统并发用户数不低于200
- 普通页面响应时间不超过2秒
- 复杂查询响应时间不超过5秒
- 系统可用性达到99.5%

### 4.2 安全需求

- 用户密码采用加密存储
- 接口访问需进行身份认证
- 敏感操作记录操作日志
- 防止SQL注入和XSS攻击

### 4.3 兼容性需求

- PC管理端兼容Chrome、Edge、Firefox等主流浏览器
- 微信小程序端兼容微信最新版本及前两个主版本

### 4.4 可维护性需求

- 代码结构清晰，遵循分层架构
- 核心配置支持外部化
- 提供完整的开发文档和部署文档

---

## 5. 外部接口需求

### 5.1 用户接口

- PC Web端：基于Vue 3的单页应用，适配1920×1080及以上分辨率
- 微信小程序端：基于Uni-App开发的微信小程序

### 5.2 硬件接口

- 服务器：支持x86_64架构的物理机或虚拟机
- 扫码设备：用户端通过手机摄像头实现ISBN扫码

### 5.3 软件接口

- 微信登录接口（OAuth2.0）
- 图书ISBN信息查询第三方API（如豆瓣API、国家图书馆API）

---

## 6. 数据需求

### 6.1 静态数据

- 图书基础信息（书名、作者、出版社、ISBN等）
- 图书分类信息
- 用户角色与权限配置
- 系统参数配置

### 6.2 动态数据

- 借阅记录
- 预约记录
- 操作日志
- 用户登录会话

## 7. 附录

### 7.1 需求跟踪矩阵

| 需求编号 | 需求描述 | 优先级 | 主要端 | 验收依据 |
|----------|----------|--------|--------|----------|
| RQ-001 | 用户登录、登出、Token刷新与当前用户查询 | 高 | 管理端/小程序端/后端 | 登录成功、失败、过期、刷新、退出场景均通过 |
| RQ-002 | RBAC 用户、角色、权限管理 | 高 | 管理端/后端 | 管理员可配置角色权限，普通用户无法访问管理接口 |
| RQ-003 | 图书分类维护 | 高 | 管理端/后端 | 支持新增、编辑、删除、树形查询，删除前校验关联图书 |
| RQ-004 | 图书信息管理 | 高 | 管理端/后端 | 支持 CRUD、上下架、封面、库存、ISBN解析失败兜底人工录入 |
| RQ-005 | 图书检索与详情 | 高 | 小程序端/管理端/后端 | 支持关键词、ISBN、分类、分页、详情展示 |
| RQ-006 | 借阅申请、审批、取书、归还 | 高 | 小程序端/管理端/后端 | 完整借阅闭环通过，库存一致 |
| RQ-007 | 续借、逾期、罚金 | 高 | 小程序端/管理端/后端 | 超期状态、续借次数、罚金计算准确 |
| RQ-008 | 预约、取消、归还后通知队列 | 中 | 小程序端/后端 | 无库存可预约，归还后按预约顺序通知 |
| RQ-009 | 公告发布与展示 | 中 | 管理端/小程序端/后端 | 管理端发布，小程序端可见 |
| RQ-010 | 数据统计与可视化 | 中 | 管理端/后端 | Dashboard 展示核心统计指标和热门排行 |
| RQ-011 | 操作日志与审计 | 中 | 管理端/后端 | 关键操作可查询、可追踪操作者和时间 |
| RQ-012 | Docker 部署与健康检查 | 高 | 运维 | `docker compose up -d --build` 后管理端和后端可访问 |

### 7.2 后续 Agent 开发约束

1. 开发任一需求前，先确认本需求编号、接口文档、数据库表和测试用例是否对应。
2. 高优先级需求未完成前，不应扩展低优先级功能，避免功能范围失控。
3. 需求实现完成后必须更新《09-项目验收目标文档》中对应检查项的状态说明。
4. 第三方 ISBN、微信登录、消息通知等外部能力必须提供 Mock 或降级方案，保证本地和测试环境可验收。
5. 涉及个人信息的功能必须遵循最小采集原则，只保存业务必需字段。
