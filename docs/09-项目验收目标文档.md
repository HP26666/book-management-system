## 1. 编写目的

本文档用于定义图书管理系统从“框架完成”到“可交付验收”的目标、范围、检查项和通过标准。后续使用 agent 自动开发时，应把本文档作为最终验收清单，并在每次迭代完成后更新对应状态和遗留问题。

---

## 2. 验收范围

| 范围 | 是否纳入 | 说明 |
|------|----------|------|
| Spring Boot 后端接口 | 是 | 包含认证、权限、图书、分类、借阅、预约、读者、公告、统计、日志 |
| Vue PC 管理端 | 是 | 包含管理员和图书管理员的后台操作 |
| Uni-App 微信小程序端 | 是 | 包含普通读者的检索、借阅、预约、个人中心 |
| PostgreSQL 数据库 | 是 | 包含表结构、索引、迁移脚本、基础数据 |
| Redis | 是 | 包含缓存、Token 黑名单或必要的分布式控制能力 |
| Docker 部署 | 是 | 后端、管理端、PostgreSQL、Redis 必须可通过 Compose 启动 |
| 小程序上架审核 | 否 | 验收到可构建、可在微信开发者工具运行；正式审核发布另列流程 |
| 第三方 ISBN 服务稳定性 | 否 | 只验收对接和降级逻辑，不验收第三方可用性 |

---

## 3. 总体验收通过标准

项目满足以下全部条件才可判定通过：

1. 高优先级需求 100% 完成，中优先级需求完成率不低于 95%，无阻塞业务闭环的缺口。
2. 无 Critical 缺陷；无未关闭的高危安全漏洞；Major 缺陷不超过 5 个且均有明确规避方案和修复计划。
3. `docker compose up -d --build` 后管理端、后端、PostgreSQL、Redis 均可正常运行。
4. 管理端、小程序端、后端接口能完成“搜索图书 → 借阅申请 → 审批 → 归还 → 库存恢复”的核心闭环。
5. 数据库迁移脚本可在空库执行成功，重复部署流程可复现。
6. 核心接口 200 并发下平均响应时间小于 2 秒，错误率小于 1%。
7. 项目文档、接口契约、数据库结构、测试记录与实际实现一致。

## 4. 功能验收目标

### 4.1 用户认证与权限

| 编号 | 验收项 | 通过标准 | 优先级 |
|------|--------|----------|--------|
| AC-AUTH-001 | 管理端账号密码登录 | 正确账号可登录，错误账号/密码有明确提示 | 高 |
| AC-AUTH-002 | 小程序微信登录 | 微信 code 登录成功；开发/测试环境可使用 Mock 登录 | 高 |
| AC-AUTH-003 | Token 刷新与登出 | Token 过期可刷新；登出后原 Token 不可继续访问受保护接口 | 高 |
| AC-AUTH-004 | RBAC 权限控制 | 普通读者访问管理端接口返回 403，管理员可授权角色权限 | 高 |
| AC-AUTH-005 | 密码安全 | 密码使用 BCrypt 等安全算法存储，接口不返回密码字段 | 高 |

### 4.2 图书与分类管理

| 编号 | 验收项 | 通过标准 | 优先级 |
|------|--------|----------|--------|
| AC-BOOK-001 | 分类树维护 | 支持新增、编辑、删除、排序和树形查询；有关联图书时禁止删除 | 高 |
| AC-BOOK-002 | 图书 CRUD | 支持新增、编辑、详情、分页查询、逻辑删除 | 高 |
| AC-BOOK-003 | ISBN 解析 | 第三方解析成功可自动填充；失败时允许人工录入 | 中 |
| AC-BOOK-004 | 封面上传 | 限制文件类型和大小，上传成功返回可访问 URL | 中 |
| AC-BOOK-005 | 库存调整 | 总库存、可借库存符合约束；每次调整记录库存日志 | 高 |
| AC-BOOK-006 | 上下架 | 下架图书不可被普通读者借阅或预约 | 高 |

### 4.3 借阅、续借、归还

| AC-BORROW-001 | 借阅申请 | 有库存且读者符合规则时可提交申请 | 高 |

| AC-BORROW-007 | 并发控制 | 多用户同时借同一本低库存图书时不会超借 | 高 |

## 10. Agent 开发检查规则

后续 agent 每次完成一个阶段时必须执行：

1. 对照本验收文档标记已完成、未完成、阻塞项。
2. 若新增或变更接口，同步更新《05-接口设计文档》。
3. 若新增或变更表字段，同步更新《04-数据库设计说明书》和迁移脚本。
4. 若影响部署，同步更新 `docker-compose.yml`、`.env.example` 和《07-部署运维手册》。
5. 若影响用户操作，同步更新《08-用户操作手册》。
6. 功能完成但无测试或无可复现验证步骤时，不得视为验收完成。
